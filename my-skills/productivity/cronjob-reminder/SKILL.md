# Cronjob 提醒任务创建规范

## 核心规则

### ✅ 正确的 Prompt 写法

```
输出以下内容（不要添加任何额外文字）：

{提醒内容}
```

**示例：**
```
输出以下内容（不要添加任何额外文字）：

该给汽车充电了 🔋
```

### ❌ 错误的 Prompt 写法

```
发送消息提醒生哥：该给汽车充电了
```

**为什么错误？**
- "发送消息"会误导子代理去尝试 iMessage/飞书/Reminders
- 系统会自动注入提示：不要使用 send_message，系统会自动投递
- 两者矛盾 → 子代理尝试其他渠道 → 失败

---

## 工作原理

1. **系统注入提示**（自动）：
   ```
   [IMPORTANT: ... DELIVERY: Your final response will be automatically delivered 
   to the user — do NOT use send_message or try to deliver the output yourself. ...]
   ```

2. **子代理行为**：
   - 正确 prompt：直接输出内容 → 系统投递到 deliver 指定的平台
   - 错误 prompt：尝试"发送消息" → 绕过系统投递 → 失败

---

## 创建任务模板

```yaml
name: {任务名称}
prompt: |
  输出以下内容（不要添加任何额外文字）：

  {提醒内容}
schedule: {时间}
deliver: wecom  # 或 telegram/feishu
```

---

## ⚠️ 一次性 vs 重复任务

**关键：`schedule` 格式决定任务类型**

| 用户说 | 正确 schedule | 错误 schedule |
|-------|--------------|--------------|
| "今晚23点提醒我" | `"2026-05-16T23:00:00"` | `"0 23 * * *"` |
| "明天早上9点提醒" | `"2026-05-17T09:00:00"` | `"0 9 * * *"` |
| "每天23点提醒" | `"0 23 * * *"` | — |

**规则：**
- **一次性任务**（"今晚"、"明天"、"下周一"）：必须用 **ISO 时间戳**
- **重复任务**（"每天"、"每周"）：用 **cron 表达式**

**为什么重要？**
- Cron 表达式（`0 23 * * *`）默认 `repeat="forever"` → 每天重复
- ISO 时间戳（`2026-05-16T23:00:00`）默认 `repeat="once"` → 执行一次

---

## 常见错误

| 错误 prompt | 后果 |
|------------|------|
| "今晚23点提醒我" + schedule=`0 23 * * *` | ❌ 变成每天重复提醒 |
| "发送消息提醒..." | 子代理尝试 iMessage/飞书，失败 |
| "用 send_message 发送..." | 违反系统规则 |
| "通知用户..." | 可能被理解成需要调用通知 API |
